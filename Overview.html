This document is the API specification for the Simple Warehouse Application.
Central classes are HandlingUnit and Location.

Some general information.

ConfigCache

ConfigCache is refreshed every 3 seconds. See ConfigCacheBean.
ConfigCache has NO entries at startup.

Configurator

Overall configuration handling. JMX and REST interface provided to handle configuration entries.
Configuration providers are DbTable and Properties. Try JConsole to manage configurations.

TimerJpaSessions

Processing of TimerJpaSessionBean1 and TimerJpaSessionBean2 is controlled via ConfigCache:

-Timer1 UP means processing will be done
-Timer1 DOWN means processing stopped
-Timer2 UP means processing will be done
-Timer2 DOWN means processing stopped
-Timer3 UP means processing will be done
-Timer3 DOWN means processing stopped

Startup state for both timer controlled beans is NO processing.

curl -X PUT -i http://localhost:8080/war/resources/ApplConfigRestService/Entry/Timer1/UP
curl -X POST -i http://localhost:8080/war/resources/ApplConfigRestService/Entry/Timer1/DOWN

curl -X PUT -i http://localhost:8080/war/resources/ApplConfigRestService/Entry/Timer2/UP
curl -X POST -i http://localhost:8080/war/resources/ApplConfigRestService/Entry/Timer2/DOWN

Sample data

Handle Simple Warehouse sample data (Locations and Handling Units)

Initialize:
curl -X PUT -i http://localhost:8080/war/resources/TopologyRestService/SampleData

Cleanup:
curl -X DELETE -i http://localhost:8080/war/resources/TopologyRestService/SampleData
